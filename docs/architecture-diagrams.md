```

分片 0           分片 1           分片 63
┌─────────┐     ┌─────────┐     ┌─────────┐
│ pending │     │ pending │     │ pending │
│ map     │     │ map     │     │ map     │
│         │     │         │     │         │
│room-0:  │     │room-1:  │     │room-63: │
│ [msg1]  │     │ [msg2]  │     │ [msg3]  │
│         │     │         │     │         │
│room-64: │     │room-65: │     │         │
│ [msg4]  │     │ [msg5]  │     │         │
└────┬────┘     └────┬────┘     └────┬────┘
     │               │               │
     └───────────────┼───────────────┘
                     │
                     ▼
            ┌─────────────────┐
            │   shard[0].mu  │
            │   shard[1].mu  │
            │      ...       │
```

```
         │                   │
```

## 八、Token Bucket 限速

```
Token Bucket 原理
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

容量: 20 tokens (突发能力)
速率: 10 tokens/秒

时间 ─────────────────────────────────────────────────▶

用户发送消息:
    │
    │ token >= 1 ✓ 允许
    ▼
┌──────────────────────────────────────┐
│ 初始: tokens = 20                      │
│ 发送1条: tokens = 19                  │
│ 发送10条: tokens = 10                  │
│                                      │
│ 1秒后: tokens += 10 → 20 (补充)      │
│ 2秒后: tokens += 10 → 20 (上限)      │
└──────────────────────────────────────┘

用户发消息太快?
    │
    │ token < 1 ✗ 拒绝
    ▼
返回: "发送频率过快"
```
