# Barrage 网关迭代升级日志总结

本文档汇总了我们在近期迭代中，对 `barrage`（高并发弹幕系统网关）进行的全部强化重构与优化内容。系统整体紧密围绕 **高并发容灾**、**内存稳定与安全防御** 以及 **通信体验标准化** 三个维度进行了全面升级。

## 一、 系统架构与容灾稳定性升级

1. **解决 Reactor 底层线程死锁漏洞（关键）**
   - 发现并彻底修复了当瞬时弹幕迸发打满业务系统时，`ants` Worker 协程池耗尽引发的调用死等问题。
   - 于 `internal/worker/pool.go` 引用了 `ants.WithNonblocking(true)`。遭遇超额冲击将果断丢弃任务以实现降级限流，坚决保证 `gnet` Event-Loop 的持续运转权，杜绝全服卡死现象。

2. **空房清理（GC）无锁异步化**
   - 原先用户离房导致房间清空时，采用原地的 `Bucket Lock` 同步删除策略，极易在高并发散场时阻塞网络 IO。
   - 现重构为 **标记延迟清除**：引入 `LastEmptyTime` 属性记录离房时间，剥离销毁操作至全新的独立后台 GC 守护协程 `RoomManager.StartRoomCleaner` 行使（例如 3 分钟轮询淘汰），完美释放了核心热点线程互斥锁并发压力。

3. **去除底层多余缓冲创建，压降内存 GC 毛刺**
   - 修复了此前每接收一次 WebSocket 数据都重复调用 `append([]byte(nil), payload...)` 产生多重废弃对象切片的问题。
   - 通过直接穿透原生引用，巨量减轻了由于不必要内存复制导致频繁引发引擎 GC STW (Stop-The-World) 停顿的隐患。

## 二、 慢连接治理与精准降级

1. **分级降级抛弃系统（Tiered Discarding）改进**
   针对网络拥堵但尚未断卡的客户端积压作了极为细腻的拦截机制：
   - **4MB 一刀切防 OOM 强退断开**：任何连接出站缓冲区越过红线，强制斩断 `c.Close()` 防止服务雪崩。
   - **VIP 用户特权保护（1.5MB 限度）**：VIP 发送缓冲放宽限制，尽最高可能保活。
   - **普通用户弹性抛弃**：
     - 若积压达大阈值（>512KB）：百分百跳过闲杂人等的大部分弹幕抛弃。
     - 若属于轻微抖动拥堵（>256KB）：引入了创新且快速的**无锁状态概率计算机制**（依据发包的时间戳 `%2` 算数进行 50% 抛弃）。既能让客户端界面不至于“死寂无弹幕”，又成功砍半了发包负载缓冲。

## 三、 内存安全边界与防攻击防护

1. **双重边界限制防止 DdoS 内存拖爆（Slowloris 防护）**
   - 于接入前端把控死线。严格限定 `HandShake` 握手阶段内存拼接最高不越 `4096 Byte`，防范恶意用户慢速无限发头。
   - `WebSocket` 长链接解析端对半包读取设置强制容许上线 `130KB` 绝杀阀值，超过立即断连。

2. **WebSocket 载荷防弹服 & Protobuf 一致约定限制**
   - 指定系统单针包界限 `MaxWebSocketPayloadSize = 64KB` 保护大体积非法数据冲死编解码框架。
   - 连同 `app.go` 中的 `HandleInbound` 二步逻辑入口一概拦截防御。

## 四、 协议规范性强化 

1. **WebSocket 标准协议边界**
   - 明确设定不支持任何碎片包续链（`ErrFragmentNotSupported`）。
   - 严格审查 RSV 保留位规范限制（不允许客户端挂伪装扩展位扰乱服务端）。
   - 对于 `Close()` 请求实现了依据 RFC 原样萃取前 16位 客户端关闭码的规约反馈，大大缓解了各类不同前端报错的偶发 Bug。

2. **操作鉴权防非法注入**
   - 控制帧限制强制不许超过定长 `125 byte`，非法 opcode 指令强硬打回断连。

## 五、 全局观测打点注入（Metrics 监控就绪）

新增了统一的 `metrics` 内存原子观测架构体系。后续只需平滑引入 Prometheus `NewCounterFunc` 拉取闭环即可无缝连接至 Grafana。现有关键遥测：
- `BroadcastDispatchCount / ErrCount`（监控底层发包派送效率）
- `KafkaPublishErrCount / KafkaConsumeErrCount`（监控集群消息管道的运行顺滑度，并配好重试与后向避让逻辑）
- `SlowConnSkipCount`（监控跳级连接和降级抛弃发生频率）
- `FilterRejectCount`（统计恶词拦截阻断效率）
- 系统直接调用 `LastActiveTime` 等也全部升级为标准线程安全的 `atomic.Int64` 跨线程取值。

## 六、 鲁棒性体系建立（完善测试用例）

目前在底层模块构建了极其刚硬的内部测试：
- ✅ **自动测试 LoopDispatcher** 的路由丢包率及空回弹拒绝情况。
- ✅ **自动测试 Broadcaster** 防回声屏蔽体系（自身发送不吃自身包）以及其正确跨 Loop 并发命中逻辑。
- ✅ **自动测试 App 集成拦截** 确保了敏感恶词能够正确组装返回反推指令包（`Op_ERROR`）警告前台应用。
